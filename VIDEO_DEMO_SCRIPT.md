# Attendance App Demo Script

## Goal
Create a 2-minute product demo that shows the core workflow of the system:
staff clock in and out, managers review approvals, admins export reports, and visitors can see the public company site.

## Target Audience
- University project panel
- Client or stakeholder review
- Short website/app showcase

## Runtime
- 2 minutes

## Core Message
This system tracks attendance with role-based access, location verification, manager approval, and reporting tools.

## Scene Plan

### Scene 1: Public Landing Page, 0:00 - 0:15
Show the public homepage and mention that the system has a branded public site with Home, Services, Contact, and Login.

On-screen action:
- Scroll the homepage slightly.
- Hover the navigation bar.
- Click through Services or Contact briefly if needed.

Narration:
"This is the public face of the system. Visitors can learn about the company and access the login page from the main navigation."

### Scene 2: Role-Based Login, 0:15 - 0:25
Show the login page and then transition into the staff or manager dashboard.

On-screen action:
- Enter a staff or manager account.
- Log in.
- Pause on the dashboard loading.

Narration:
"After login, users are redirected based on role. Staff go to their own panel, while admins and managers are sent to the admin side."

### Scene 3: Staff Clock-In Flow, 0:25 - 0:55
Show the staff clock-in widget with location capture and clock-in button.

On-screen action:
- Highlight the clock-in widget.
- Click Get Location.
- Show that location is captured.
- Click Clock In.

Narration:
"Staff can clock in directly from their dashboard. The system captures location information and supports verification so attendance is tied to a real work site."

Callouts:
- Location capture
- Clock-in status
- Pending approval state

### Scene 4: Staff Attendance Summary, 0:55 - 1:10
Show the staff stats widgets and the attendance logs page.

On-screen action:
- Scroll to the summary cards.
- Open attendance logs.
- Show today, week, and month totals.

Narration:
"Employees can review their own attendance history, see worked time, overtime, and track current shift status."

### Scene 5: Manager Approval Workflow, 1:10 - 1:35
Switch to a manager account and show the approvals queue or overview page.

On-screen action:
- Open Clock-In Approvals.
- Select a pending record.
- Add approval notes.
- Approve or reject the attendance.

Narration:
"Managers review pending and temporary attendance records, then approve or reject them with notes. This keeps the workflow controlled and auditable."

Callouts:
- Pending approvals
- Approval notes
- Staff attendance overview

### Scene 6: Admin Reporting, 1:35 - 1:50
Show the admin attendance table and export options.

On-screen action:
- Open Attendance Logs.
- Show filters and table columns.
- Trigger CSV export or print view.

Narration:
"Admins get the full attendance dataset with filters and export tools, making reporting and compliance straightforward."

### Scene 7: Closing Summary, 1:50 - 2:00
Return to the dashboard or public home page with a clean end frame.

On-screen action:
- Show the main dashboard or branded landing page.
- End on a simple title card.

Narration:
"In one system, the app handles attendance capture, verification, approval, and reporting for staff, managers, and administrators."

## Suggested On-Screen Title
Attendance Tracking System Demo

## Suggested Ending Card
Role-based attendance. Location verification. Manager approvals. Exportable reports.

## Recording Notes
- Use a clean browser window and keep the cursor movement deliberate.
- Record with notifications disabled.
- If possible, use test data so the approval flow is visible.
- Keep each scene short; the demo should feel fast and confident.
